介紹

決定要使用新的技術和資源來建置個人部落格時，看了不少文章，但花了一天的時間卻總是遇到問題，像是文章中沒有說明清楚在哪個資料夾中進行指令導致沒看到結果，另外Github Page預設<username>.github.io的repo是個人網站，但不少作者推薦另外開一個repo來分開原始碼和產生的網站文檔，因此在Travis的設定檔又有所不同，以及後來可能會增加的域名調整，重做好多次，各式各樣的問題也遇到瓶頸，不死心地想找更多文章來參考，終於讓我找到這篇當參考，終於搞定！來寫下心得，幫助未來的自己，也給需要的朋友。

為什麼一定要用這個方式？

簡單介紹

Hugo

Hugo它是開源的靜態站點生成器框架，即是基於簡單的文本內容即可生成可以直接在瀏覽器訪問的 html 網頁，對於Hugo來說，文本內容是採用 Markdown 語法的 md 來源文件，採用GO語言編寫，優點是執行效率高。

Github Pages

Github提供的免費的靜態網站托管服務，使用方便而且功能強大，雖然有限制但也夠用，還可以綁定自己的域名。

Travis CI

Travis CI 是一個持續集成服務，用於構建和部署專案。什麼是持續集成呢？即是當專案代碼與之前不同時會自動構建和部署，確保專案維持最新的內容運行。Travis CI 在 Github 上的開源項目上有提供免費的服務。

準備

  • Hugo Install
  • GitHub account
  • Travis CI 帳號，可通過GitHub帳號登入

GitHub Pages 限制

GitHub Pages 站點在使用上有如下限制：

  1. 倉庫大小不得超過1GB；
  2. 建置的 GitHub Pages 站點大小不得超過1 GB；
  3. GitHub Pages 站點每月限 100 GB 流量；
  4. GitHub Pages 站點每小時限構建10次；

更多介紹見官方文檔 「What is GitHub Pages?」

2. 部署Hugo Site 到 GitHub Pages

上傳到Github之前，先在Github中添加一個空白repository，註意不要添加如README，.gitignore等文檔。因為我是使用默認的<username>.github.io由此得到Github中該repository的網址：https://github.com/<username>/vnevrz.github.io

  • 先把原始碼提交到 GitHub 的一个 repo ( 原始碼repo —> vnevrz.github.io)

    1
2
3
4

    git add -A
git commint -m "initial hugo files"
git remote add origin https://github.com/<username>/vnevrz.github.io
git push -u origin master

  • config.toml 的 baseURL 要設置成 https://.github.io

  • 生成之後部署要用的 Github Access Token，至少要有 public_repo 的權限。（別勾有private的那個選項），参考 這邊 来完成此操作，一定要記下來Token值，因為之後都不會顯示了。在Github的Developer settings頁面中生成personal access tokens，若是私有倉庫，勾選repo；若是公開倉庫，勾選public_repo。

  • 假設想要分開網站原始碼及產生的靜態文件倉庫，可參考這篇。

3.利用Travis CI 自動部署

  • 生成的token須在Travis CI的目標Repo中設置，到setting頁面中會看到所有的Repo，將要用來的部屬的網站點ON。

  • 確保General中的 Build pushed branches, Build pushed pull requests已啓用。官方文檔GitHub Pages Deployment定義的默認token名爲GITHUB_TOKEN，其它指令詳見文檔。

  • 這時定義的相關變量，之後會在建置時寫入.travis.yml文件。

回到bash中，在一開始創建的站台 repo（我是Hugome）中在根目錄增加 .travis.yml

1

$ touch .travis.yml

sudo: false

deploy:
	provider: pages
	skip_cleanup: true
	github_token: $GITHUB_TOKEN
	on:
		branch: master
	local_dir: public
	repo: <username>/<username>.github.io
	fqdn: <custom-domain-if-needed>
	target_branch: master
	email: <github-email>
	name: <github-username>
  • 默認情況下，travis 會自動下載 git submodules
  • github_token: $GITHUB_TOKEN 要和 travis 設置的環境變量名一致
  • fqdn: 如果需要設置自定義功能變數名稱，可以在這里設置，travis 會自動生成 CNAME 文件提交，同時要設置 config.toml 中的相應的 baseURL，關於如何配置個人域名，可以参考 這篇文章 # 配置个人域名

最後，可以手動去 travis 觸發一次 build 檢查效果。如果設置了提交觸發 build，之後每次目標 repo 有提交都會自動 build，不再需要觀看travis 狀態。
