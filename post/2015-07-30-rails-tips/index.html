###拿到一個 API repo時可能需要做的### 1. 補上database相關的yml檔 2. 確認資料庫名稱及密碼都填正確之後，bundle install 3. 因為有舊的資料，所以把需要更新的table刪除，然後import新的檔，接著rake db:migrate

接著遇到的問題，然後發現是ST3中yml檔副檔名錯誤 又遇到問題，是因為在新import資料庫之前已經migrate，但不完整到一半被abort，所以會被以為資料表重複 因此下指令 rake db:rollback 仍是遇到重複的問題，所以進到mysql直接import本來的舊資料 ，先把主要的先import後rake db:migrate，刪除本來的articles，import新建的articles， 終於可以跑起server了，要注意port設定，所有檔案的路徑都要改過
